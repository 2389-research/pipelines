# local-gen-test

Throwaway working directory for testing the enrich → ledger → local-gen flow on a small set of sprints.

## Layout

```
local-gen-test/
├── inputs/
│   └── spec.md              ← put your product spec here (read by enrich step)
└── .ai/
    └── sprints/
        ├── SPRINT-001.md    ← thin sprint docs go here (zero-padded ids)
        ├── SPRINT-002.md
        └── ...
```

After the runbook below executes, this directory will also contain:

```
.ai/
├── ledger.tsv               ← generated by tools/init-ledger.sh
├── current_sprint_id.txt    ← written each iteration by the runner
├── last_test_output.txt     ← written by RunTests
├── last_audit_output.txt    ← written by Audit (if it fails)
├── local_fix_attempts       ← per-sprint LocalFix counter
└── local_exhausted          ← flag file when local model gives up
```

Plus whatever code the sprints generate (Go / TS / Python files at the paths each sprint specifies).

## Prereqs

1. **Ollama running** with `qwen3.6:35b-a3b-q8_0` pulled — `ollama pull qwen3.6:35b-a3b-q8_0`
2. **API keys configured** — `tracker setup` (Anthropic for enrichment + failure summary, OpenAI for CloudFix)
3. **Build tooling for the target language** — `go`, `node`, or `uv` depending on what the sprints target. The pipelines auto-detect.
4. **Thin sprint docs** dropped in `.ai/sprints/` with zero-padded ids (`SPRINT-001.md`, not `SPRINT-1.md`).
5. **Spec** dropped at `inputs/spec.md`.

## Runbook

All commands run from this directory:

```bash
cd /Users/michaelsugimura/Documents/GitHub/pipelines/experiments/local-gen-test

# 1. Sonnet enriches every thin sprint in place (cloud cost: ~1 call per sprint)
tracker ../../enrich_runner.dip

# 2. Eyeball at least the first enriched sprint before going further
less .ai/sprints/SPRINT-001.md

# 3. Generate the ledger from the now-enriched sprints
sh ../../tools/init-ledger.sh

# 4. Run the multi-sprint local-gen loop ($0 happy path; cloud only on local-exhaustion)
tracker ../../sprint_runner_local_gen_qwen.dip
```

## Common adjustments

- **Already shipped some sprints?** Edit `.ai/ledger.tsv` and change column 3 from `planned` to `completed` for the ones already done. The runner will skip them.
- **Want to redo enrichment for one sprint?** Delete its `## Interface contract` section (or just overwrite the file with the thin version) and rerun `enrich_runner.dip` — it'll only touch sprints missing that section.
- **Want to test single-sprint exec instead of the loop?** Run `tracker ../../sprint_exec_local_gen_qwen.dip` after enriching — it picks up the first sprint doc it finds.

## Cleanup between runs

```bash
rm -rf .ai/ledger.tsv .ai/current_sprint_id.txt .ai/local_fix_attempts .ai/local_exhausted .ai/last_test_output.txt .ai/last_audit_output.txt
git checkout -- .   # if this directory is itself a git repo
```

If you want to start completely fresh (re-enrich from thin sprints), keep your `inputs/spec.md` and the *original thin* sprint files somewhere safe — once enriched, they're overwritten in place.
