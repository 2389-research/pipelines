# validate_output error-token taxonomy

This file is the **single source of truth** for what every `validate_output`
error token means and how to fix it. `RewriteOutput` (and any future
auto-recovery agents) should read this file and apply the mappings literally.

**Do NOT search the codebase for the validator script.** The tokens themselves
fully describe the fix. The validator is `dr/spec_to_sprints.dip:validate_output`
if you absolutely need to verify a token's exact emission point, but the
recipes below are authoritative.

---

## Token format

Most tokens have the shape:

    <category>-<file-basename>

where `<file-basename>` is e.g. `SPRINT-005.yaml` or `SPRINT-012.md`. A few
tokens are bare (no file basename) — those refer to global state (ledger,
project layout).

`validate_output`'s full stdout starts with `invalid-` and contains a
space-separated list of tokens, e.g.:

    invalid-missing-scope-SPRINT-027.md missing-requirements-SPRINT-028.md ledger-md-mismatch

When iterating, process tokens one at a time. Apply the fix recipe. Re-run
validation. Stop when you get `valid`.

---

## Recipes by category

### 1. Missing MD sections — `.md` files

Generated by the section-grep loop. The file exists but is missing a required
H2 heading.

| Token | Section the file is missing | Fix |
|---|---|---|
| `missing-scope-SPRINT-NNN.md` | `## Scope` | Add a `## Scope` section to `.ai/sprints/SPRINT-NNN.md`. Pull 2-3 sentences from the original sprint plan or the spec analysis. |
| `missing-requirements-SPRINT-NNN.md` | `## Requirements` | Add `## Requirements` listing FR IDs covered (e.g. `- FR12: returning user detection`). |
| `missing-dependencies-SPRINT-NNN.md` | `## Dependencies` | Add `## Dependencies` listing sprint numbers that must complete first, or `- None`. |
| `missing-expected-artifacts-SPRINT-NNN.md` | `## Expected Artifacts` | Add `## Expected Artifacts` with specific file paths. |
| `missing-dod-SPRINT-NNN.md` | `## DoD` | Add `## DoD` as a checkbox list (`- [ ] item`). |
| `missing-validation-SPRINT-NNN.md` | `## Validation` | Add `## Validation` with exact commands (e.g. ```- `npx vitest run` → exit 0```). |

**How to patch:** read the existing MD, append the missing section in the right
place (preserve the canonical heading order shown in
`write_sprint_docs`'s prompt: Scope → Non-goals → Requirements → Dependencies
→ Expected Artifacts → DoD → Validation).

### 2. Missing YAML top-level fields — `.yaml` files

Same `missing-<field>-SPRINT-NNN.yaml` shape, but the file is **yaml** so
patch with `yq -i`.

| Token | Field | Fix recipe |
|---|---|---|
| `missing-id-SPRINT-NNN.yaml` | `id` | `yq -i '.id = "NNN"' .ai/sprints/SPRINT-NNN.yaml` |
| `missing-title-SPRINT-NNN.yaml` | `title` | `yq -i '.title = "<sprint title>"' …` |
| `missing-status-SPRINT-NNN.yaml` | `status` | `yq -i '.status = "planned"' …` |
| `missing-dod-SPRINT-NNN.yaml` | `dod` (list of strings) | `yq -i '.dod = ["DoD item 1", "DoD item 2"]' …` |
| `missing-validation-SPRINT-NNN.yaml` | `validation.commands` (list) | `yq -i '.validation.commands = [{"cmd": "npx vitest run", "expect": "exit_0"}]' …` |
| `missing-scope-fence-SPRINT-NNN.yaml` | `scope_fence.off_limits` | `yq -i '.scope_fence.off_limits = ["do not modify .ai/ledger.yaml"]' …` |
| `missing-artifacts-SPRINT-NNN.yaml` | `artifacts.creates` | `yq -i '.artifacts.creates = [{"path": "src/...", "type": "module"}]' …` |

### 3. Missing or null stack subfields

| Token | Field | Fix |
|---|---|---|
| `missing-stack-lang-SPRINT-NNN.yaml` | `stack.lang` is absent | `yq -i '.stack.lang = "javascript"' …` (pull canonical value from `.ai/spec_analysis.md` Tech Stack section) |
| `missing-stack-runner-SPRINT-NNN.yaml` | `stack.runner` is absent | `yq -i '.stack.runner = "npm"' …` |
| `missing-stack-test-SPRINT-NNN.yaml` | `stack.test` is absent | `yq -i '.stack.test = "vitest"' …` |
| `missing-stack-lint-SPRINT-NNN.yaml` | `stack.lint` is absent | `yq -i '.stack.lint = "eslint"' …` |
| `missing-stack-build-SPRINT-NNN.yaml` | `stack.build` is absent | `yq -i '.stack.build = "npm run build"' …` (or `null` if no build step) |
| `null-stack-<field>-SPRINT-000.yaml` | The bootstrap sprint (SPRINT-000.yaml) has a null stack subfield. Bootstrap sprints MUST have concrete values, not null. | Same as above — set a real value from `.ai/spec_analysis.md`. |

### 4. Structural errors — bare tokens

These don't have a file basename; they reference global state.

| Token | Meaning | Fix |
|---|---|---|
| `sprint-files-in-root` | Sprint files (`SPRINT-*.md`, `SPRINT-*.yaml`) exist in the project root instead of `.ai/sprints/` | `mv SPRINT-*.md SPRINT-*.yaml .ai/sprints/` |
| `ledger-yaml-mismatch` | The IDs in `.ai/ledger.yaml` don't match the `.yaml` files in `.ai/sprints/` | Regenerate `.ai/ledger.yaml` from the sprint YAMLs on disk. The `write_ledger` tool does this; run its body manually if needed. |
| `ledger-md-mismatch` | The IDs in `.ai/ledger.yaml` don't match the `.md` files in `.ai/sprints/` | Same fix — regenerate the ledger from the canonical YAML/MD set. |
| `empty-ledger` | `.ai/ledger.yaml` has no `sprints:` entries | Same — regenerate from the SPRINT-*.yaml files. |
| `insufficient-bootstrap-sprints` | Fewer than 2 sprints have `bootstrap: true` | Mark the first 2-3 sprints (000, 001, 002) as `bootstrap: true` via `yq -i`. |
| `invalid-yaml-SPRINT-NNN.yaml` | `yq` failed to parse the file | Open `.ai/sprints/SPRINT-NNN.yaml` and fix the syntax error (usually indentation or quoting). |
| `oversized-sprint-SPRINT-NNN.yaml-K-dod-items` | The sprint has K > 15 DoD items, which signals scope breadth | Trim the `dod` list down. Don't redecompose from inside RewriteOutput — that's the runner's job. Pick the most essential DoD items and drop the marginal ones. |
| `missing-sprint-yaml-NNN` | A redecompose-result references sprint NNN but the file `.ai/sprints/SPRINT-NNN.yaml` is missing | Write the missing sprint YAML using the standard template (see `write_sprint_docs` for the format). |

## Heuristics

1. **The error token IS the fix instruction.** If you find yourself searching
   for "where is the validator defined" or grepping for the error string in
   the codebase, you are off-track. Stop and re-read the token.

2. **Process tokens in batches by category.** All `missing-scope-*` tokens are
   the same shape of fix — write a single script that loops over them rather
   than handling them individually.

3. **Always re-read the file before patching.** Earlier turns may have already
   fixed something; don't double-apply.

4. **Verify with the validator's own commands.** Each YAML check uses
   `yq -e '.path.to.field' file` — you can run the same command to confirm a
   fix landed before claiming success.

5. **Never modify the ledger directly to make `*-mismatch` errors disappear.**
   Always regenerate it from the per-sprint YAMLs (which are the source of
   truth). The mismatch usually means YAMLs were written without updating the
   ledger; fix the ledger, not the YAMLs.
