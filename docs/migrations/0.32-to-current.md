# Migration: dippin v0.32 → current toolchain

Version-by-version notes for downstream integrators who track this repo's
`.dip` files as templates. Each entry lists what landed and what (if anything)
you need to do to adopt it.

## Current pins

- **README requirement:** tracker ≥ `v0.35.0` (vendors dippin-lang ≥ `v0.35.0`;
  joint release, tracker tagged first) — see [`../../README.md`](../../README.md).
- **Toolchain this repo was last verified against:** dippin `v0.43.0`,
  tracker `v0.35.1` (`dippin --version` / `tracker --version`).

The features below are additive; a node that does not use a field is unaffected.
Where a field is silently ignored on older releases, that is called out so you
know to bump the pin rather than expect a hard error.

## v0.32 — `tool_access: none`

Language-level field on `agent` nodes that strips the model's entire tool
catalog. Replaces "do not use tools" prose with a structural bound. Closed
[dippin-lang#41](https://github.com/2389-research/dippin-lang/issues/41).
Full detail and the prose-vs-policy distinction are in
[`0.32-tool-access-none.md`](./0.32-tool-access-none.md).

Compatibility: tracker < v0.31.0 silently ignores the field.

## v0.33 — `command_file:`

`tool` nodes can point at an external script via `command_file:` instead of an
inline `command:` block. The path is **relative** to the `.dip` file (no
absolute paths, no symlinks, no parent-tree escape); tracker inlines the body
into the packed `.dipx` at build time, so the script must be self-contained
(`scripts/lib/` is not shipped). Used throughout
[`../../dev_loop/dev_loop.dip`](../../dev_loop/dev_loop.dip)
(`command_file: scripts/setup_run.sh`, etc.).

Also in this window: tracker gained markdown-emphasis tolerance when parsing
`STATUS` lines.

## v0.34 — `prompt_file:` / `system_prompt_file:`

`agent` nodes can externalize their prompt and system prompt to files via
`prompt_file:` and `system_prompt_file:` instead of inline `prompt:` /
`system_prompt:` blocks. Same relative-path rules as `command_file:`. See the
issue-selection and squad agents in
[`../../dev_loop/dev_loop.dip`](../../dev_loop/dev_loop.dip)
(`system_prompt_file: prompts/select_next_issue.system.md` +
`prompt_file: prompts/select_next_issue.task.md`).

## v0.35 — `writable_paths:`, DIP140, per-branch overrides

The current pin floor. Adds:

- **`writable_paths:`** — a filesystem-jail bound on agents that need *scoped*
  write access. Unlike `tool_access: none` (all-or-nothing), this keeps the tool
  catalog but confines writes to the listed path globs. Used in
  [`../../greenfield/greenfield_review.dip`](../../greenfield/greenfield_review.dip)
  (`writable_paths: workspace/.review-failed`) and the implementer node in
  [`../../dev_loop/dev_loop.dip`](../../dev_loop/dev_loop.dip)
  (`writable_paths: .dev_loop_worktree/**`).
  - **Enforcement is Linux-only** (Landlock + openat2). On macOS/Windows tracker
    **refuses to start** when the field is set. Full-chain symlink rejection is
    enforced.
- **DIP140** — lint that flags params-based bypass of the `writable_paths` jail.
- **Per-branch `writable_paths` overrides** — parallel branches can declare
  their own jail scope. DOT round-trip of the field was also fixed.
- **tracker `validation_overridden` terminal** — an explicit terminal state for
  runs that proceed past a failed validation gate.

Compatibility: older tracker releases reject `writable_paths` as an **unknown
field** (hard error, not silently ignored) — this is why the README pins
tracker ≥ v0.35.0.

## v0.36 — DIP143 subgraph containment

- **DIP143** — an advisory lint for subgraph containment (a subgraph's nodes
  should stay within its declared boundary). Background and this repo's audit are
  in [`../dip143-subgraph-audit.md`](../dip143-subgraph-audit.md).
- **Leaf-TOCTOU hardening** on the `writable_paths` jail check.

## Beyond v0.36

dippin has continued past v0.36 (this repo is verified against `v0.43.0`); those
releases are additive and have not required changes to the `.dip` files or to the
fields documented here. The README pin remains tracker ≥ v0.35.0 because that is
the floor at which every field used in this repo is recognized and enforced.
