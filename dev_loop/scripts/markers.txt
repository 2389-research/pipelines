# dev_loop marker registry.
#
# Every line below is a literal marker that one of dev_loop/scripts/*.sh
# may emit on stdout for tracker to consume as ctx.tool_marker. Each marker
# is referenced by exactly one marker_grep regex on a routing tool node in
# dev_loop/dev_loop.dip. tests/test_marker_coverage.sh asserts the union of
# all markers a script COULD emit is a subset of this file, and that every
# marker_grep regex in dev_loop.dip matches at least one entry below.
#
# Format: one literal marker per line. Comments start with #. Blank lines
# allowed. Variable-iter markers are listed with the placeholder convention
# documented inline (e.g. iter-init-1-of-5).
#
# When you add a new script branch, add the marker here FIRST, then update
# the corresponding marker_grep on the routing node.
